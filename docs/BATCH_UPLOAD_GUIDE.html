📦 Batch Product Upload - Quick Reference Guide

🚀 Quick Start

  1. Navigate to Inventory → Batch Upload
  2. Download the CSV template
  3. Fill in your products (delete example rows)
  4. Upload the file and process

📍 Access Points

Method 1: From Inventory Index Page
views/inventory/index.php → Click "Batch Upload" button (blue button with CSV icon)
Method 2: From Create Product Page
views/inventory/create.php → Click "Batch Upload" button (top right)
Direct URL:
http://localhost/ella-pos/views/inventory/batch_upload.php

⚙️ Upload Options

Skip Duplicates (Recommended)

✅ Checked by default. If a product with the same SKU or Barcode already exists, it will be skipped and reported in warnings.

Update Existing

If enabled, products with matching SKU/Barcode will have their prices updated, and initial_stock will be ADDED to current stock.

✅ Validation Rules

  • ✓ Product name must not be empty
  • ✓ Category ID must be a valid number and exist in database
  • ✓ At least one price (capital OR retail) must be provided
  • ✓ If only capital price provided, retail = capital
  • ✓ If only retail price provided, capital = retail
  • ✓ Empty rows are automatically skipped
  • ✓ Rows starting with # are treated as comments and skipped

🔍 Duplicate Detection

The system checks for duplicates based on:

  • SKU - If SKU is provided and matches existing product
  • Barcode - If Barcode is provided and matches existing product
⚠️ Note: If neither SKU nor Barcode is provided, the system cannot detect duplicates and will create a new product even if the name is identical to an existing one.

📊 Processing Flow

  1. File is uploaded and validated (format check)
  2. CSV is parsed, headers are extracted
  3. Each row is validated for required fields
  4. Duplicate check (if SKU/Barcode exists)
  5. For new products:
    • Insert into products table
    • Insert into product_variations table
    • Insert into inventory table (if stock > 0)
    • Create stock_movements record
  6. For updates (if update mode enabled):
    • Update product_variations prices
    • Add to existing stock
    • Create stock_movements record
  7. Return summary with success/skip/error counts

📈 Results & Feedback

Success Message

✔ Batch upload completed: X products added/updated, Y skipped, Z errors

Warnings Section

Lists each problematic row with specific reason:

  • Row 5: Product name is required, skipped
  • Row 8: Duplicate found ('Existing Product Name'), skipped
  • Row 12: Valid category ID is required, skipped

🎯 Use Cases

Update existing: ❌ OFF

💡 Tips & Best Practices

  • ✓ Always download and use the template to ensure correct format
  • ✓ Delete the example rows and instruction rows before uploading
  • ✓ Use unique SKU codes for reliable duplicate detection
  • ✓ Test with a small batch (5-10 products) first
  • ✓ Review the category IDs in the template or upload page
  • ✓ If you see warnings, fix them and re-upload
  • ✓ Keep a backup of your original CSV file
  • ✓ For Excel files (.xlsx), ensure PhpSpreadsheet is installed or convert to CSV

⚠️ Excel Support Requirement

For .xlsx and .xls file support, you need to install PhpSpreadsheet:

composer require phpoffice/phpspreadsheet

Without this library, you must convert Excel files to CSV format before uploading.

🔐 Security & Permissions

  • ✓ User must be logged in
  • ✓ Requires role: Admin, Manager, or Stockman
  • ✓ All stock movements are logged with user ID
  • ✓ Transaction-based processing prevents partial data corruption
